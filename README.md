# 감정 기반 맛집 추천 시스템


## 프로젝트 소개

이 프로젝트는 사용자가 입력한 감정 문장을 기반으로 감정을 분석하고 그 결과에 맞는 음식과 카테고리를 추천한 뒤 관련 맛집 정보를 제공하는 서비스입니다.

백엔드는 FastAPI를 기반으로 구성했으며 사용자가 입력한 데이터를 AI 모델에 전달하고 추론 결과를 JSON 형태로 반환해 웹 화면에 출력되도록 구현했습니다.  
또한 감정 분석 결과를 기반으로 음식과 카테고리를 추천하는 CBF(Content-Based Filtering)을 이용하여 사용자가 선택한 음식 카테고리를 바탕으로 맛집 추천 결과를 확인할 수 있도록 전체 흐름을 연결했습니다.  
최종 추천 결과는 카카오 지도 API와 연동하여 사용자가 맛집 위치를 직관적으로 확인할 수 있도록 구성했습니다.

## 내가 한 일

- FastAPI 기반으로 결과 조회용 GET API와 사용자 입력 처리용 POST API를 구현했습니다.
- 사용자가 감정 문장을 입력하면 POST 요청으로 데이터를 받아 AI 모델 추론 결과가 JSON 형태로 반환되고 화면에 출력되도록 연결했습니다.
- GET 요청을 통해 감정 분석 및 추천 결과를 확인할 수 있도록 응답 구조를 구성했습니다.
- KoBERT 감정 분석 결과를 기반으로 음식과 카테고리를 추천하는 CBF(Content-Based Filtering)로직을 구현했습니다.
- 추천 데이터를 프론트엔드에서 바로 사용할 수 있도록 응답 포맷을 정리했습니다.
- 카카오 지도 API를 연동해 추천 맛집의 위치를 지도에서 시각적으로 확인할 수 있도록 구현했습니다.

## 기술 스택

- **Backend**: FastAPI
- **AI / Recommendation**: KoBERT, CBF, NCF
- **Frontend**: React
- **Data**: Pandas, AIHub, Kaggle Food Choices, 카카오맵 크롤링
- **API**: Kakao Map API

## 핵심 기능

### 1. 감정 입력 및 분석
사용자는 웹 화면에서 감정을 직접 입력하거나 버튼을 통해 감정을 선택할 수 있습니다.  
입력된 값은 FastAPI 서버로 전달되며 AI 모델이 감정을 분석한 뒤 결과를 반환합니다.

### 2. 감정 분석 결과 출력
감정 분석 결과는 JSON 형태로 반환되며 프론트엔드에서 이를 받아 화면에 출력합니다.  
사용자는 자신의 감정 상태와 함께 추천에 필요한 결과값을 확인할 수 있습니다.

### 3. CBF 기반 음식/카테고리 추천
KoBERT 감정 분석 결과를 입력으로 받아 현재 감정에 맞는 음식과 카테고리를 추천합니다.  
추천 결과는 사용자 선택이 가능하도록 화면에 표시됩니다.

### 4. 맛집 추천 결과 제공
사용자가 선택한 음식 카테고리를 기반으로 추천 맛집 정보를 출력합니다.  
맛집 이름, 주소, 평점 등의 정보를 화면에서 확인할 수 있습니다.

### 5. 카카오 지도 API 연동
추천된 맛집의 위치를 카카오 지도 API와 연동해 지도에서 시각적으로 확인할 수 있도록 구현했습니다.
